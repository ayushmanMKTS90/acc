How to Use — Account Intel

What This Is

Account Intel generates a 7-layer pre-meeting brief for any B2B target account. It combines a local LLM (gemma3:4b via hosted Ollama), live industry news (GNews API), and automated tech-stack detection (technology-detector npm package) to produce a complete intelligence report in about 60 seconds. The report is rendered as an HTML dashboard with full-width rows, layered insight sections, and an animated loading sequence.

Step-by-step

  • 1Pick an Industry — signals and pain points auto-fill from presets
  • 2Type the Company name and Domain (needed for tech detection)
  • 3Enter your Product — what you sell (optional, not pre-filled)
  • 4Set Role and Seniority of the prospect
  • 5Tweak the auto-populated signals if you know more about the account
  • 6Click Generate Brief — takes about 60 seconds
  • 7Report opens in a new tab; auto-saves to Saved Briefs (stored as JSON files in the GitHub repo)
  • 8Inside the report, click Save as ICP Profile to store the form state
  • 9Apply a saved ICP profile later to pre-fill the form for similar accounts

Report Sections (7 Layers)

  • 1Why this account — Buying signals, pressure events, validation questions, and confidence score. Tells you why they're likely to buy now.
  • 2Industry context — Live headlines from GNews, curated talk tracks, and vertical shift recommendations.
  • 3Who you're meeting — Their priorities, concerns, influence level, and a tailored success vision.
  • 4Tech stack — Detected via technology-detector scanning the domain. Falls back to industry signals if no domain is given.
  • 5How to position — Reframe against competitors, leverage points, and phrases to avoid.
  • 6Conversation path — Best lead path, fallback angle, trigger moment, value sequence, and call flow steps.
  • 7Discovery guide — Validation questions, cues to listen for, pivot topics, and success criteria.

Architecture — Execution Order & Context Flow

Layers execute in 3 parallel waves then a final step. Upstream layers pass relevant context downstream so each layer builds on the last.

Wave 1
L1 — Signals · L2 — Industry (GNews) · L4 — Tech Stack
Run in parallel. L1 produces summary, pressure events, inferred signals. L2 fetches live news via GNews with 1hr per-industry cache. L4 runs technology-detector against the domain (or falls back to industry-preset signals with ≥3 matches = medium confidence).
Passes downstream: L1 outputs → L3, L5, L6. L4 outputs → L5, L6.
Wave 2
L3 — Prospect · L5 — Compete
Run after Wave 1 completes. L3 receives L1's summary and pressure context to assess concerns, priorities, and influence level (based on role/seniority, not default "decision-maker"). L5 receives both L1 and L4 context for competitive reframing and leverage identification.
Passes downstream: L3 outputs → L6. L5 outputs → L7.
Wave 3
L6 — Orchestrate
Runs after Wave 2. Receives L1 + L3 + L4 context to build the conversation path with best lead path, backup fallback, trigger plan, value proposition sequence, and call flow steps.
Passes downstream: All L6 outputs → L7.
Final
L7 — Discovery
Receives the concatenated outputs of all 6 upstream layers. Uses an 8k token budget (vs 3k for other layers). Generates validation questions, cues to listen for, pivot topics if they push back, phrases to avoid, and success criteria.
Input: All 6 layer outputs concatenated.

API Endpoints

POST/api/brief-htmlMain endpoint. Orchestrates all 7 layers and returns an HTML dashboard report. Accepts company, domain, industry, product, role, seniority, signals, painPoint (dropdown: lead generation, pipeline velocity, competitive displacement, differentiation, security compliance, cost reduction, digital transformation, data integration, scalability, vendor consolidation, roi justification, change management, or custom). Serverless maxDuration: 120s.

Configuration & Dependencies

LLMgemma3:4b via ollama.com/api/chat. API key set via OLLAMA_API_KEY env var. Default token budget: 3,000 (L7 uses 8,000).
NewsGNews free tier (100 req/day). API key set via GNEWS_API_KEY. Results cached per industry for 1 hour in-memory.
Techtechnology-detector ^1.0.4 npm package. Scans the target domain for HTTP headers and technology fingerprints.
Nodev24.x (Vercel runtime). Engine pinned in package.json.
FontDM Sans via Google Fonts.

Useful Details

  • Product field is never pre-filled — you must enter it each time to ensure the brief is relevant to what you're selling.
  • ICP Profiles are stored as JSON files in the icps/ folder of the GitHub repo. They save the form state (industry, signals, pain point, role, seniority) so you can apply them instantly for similar accounts.
  • Saved Briefs are stored as JSON files in the briefs/ folder of the GitHub repo. Each brief stores the full report data so you can reopen or review past reports.
  • Tech detection fallback: If no domain is provided, Layer 4 matches against INCUMBENTS_SIGNALS in _config.js (8 industries mapped to likely tech signals). Requires ≥3 matches for medium confidence, ≥2 for low, otherwise "Unknown / None".
  • Influence level in Layer 3 is based on the prospect role and seniority fields — there is no default "decision-maker" label.
  • Cross-layer context is concatenated into the LLM prompt for downstream layers. L1 (summary + pressure + inferred signals) feeds L3, L5, L6. L4 (detected tech + evidence) feeds L5, L6. All layers concatenated feed L7.
  • V1 fallback — The /v1/ route serves the previous stable build. Accessible at account-intel-kappa.vercel.app/v1/.
  • Product is intentionally NOT pre-filled when selecting an industry — this is by design to prevent stale product entries.
